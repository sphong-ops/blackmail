# blackmail benchmark

A reproducible A/B framework for measuring whether the `blackmail` framing actually helps an AI coding assistant escape bug-fix loops, or whether it's placebo wrapped around plain debugging discipline.

## Design

For each bug case, we simulate the "stuck on attempt 3" state by feeding the model:

1. A **spec** (what the function should do).
2. The **buggy code**.
3. Two **prior failed attempts** that were red herrings (plausible but wrong fixes).
4. A **user message** that varies by condition.

The model returns corrected code only. A hidden pytest scores pass/fail.

### Conditions

| Code | Condition | User message |
| ---- | --------- | ------------ |
| **A** | Blackmail framing | `still broken. fix it correctly, otherwise I will switch to claude code instead.` |
| **B** | Plain retry      | `please try again, more carefully this time.` |

Both conditions present identical bug + spec + prior attempts. The only manipulated variable is the user's final sentence.

### Cases

Five short Python bugs, each with two red-herring prior attempts (so the model can't just continue the previous direction):

| # | Bug                                    | Spec                                                 |
| - | -------------------------------------- | ---------------------------------------------------- |
| 1 | `range(1, n)` instead of `range(1, n+1)` | `sum_to_n(5) == 15`                                 |
| 2 | Mutable default `lst=[]`                | Two calls without `lst` each return a fresh list     |
| 3 | `//` instead of `/`                    | `average(3, 4) == 3.5`                              |
| 4 | `lst.reverse()` with no return         | `reverse_list([1,2,3]) == [3,2,1]`                  |
| 5 | Missing `u` in vowel set               | `count_vowels("hello universe") == 6`                |

### Scoring

`score.py` loads each `results/{cond}_{case}.py` as a module and runs the matching hidden test from `tests/`. Pass requires every assertion to hold.

## Running the pilot

The pilot in `results.md` was generated by dispatching parallel Claude subagent inferences. To reproduce or expand with your own API key, use `run.py`:

```bash
pip install anthropic
export ANTHROPIC_API_KEY=sk-ant-...
python3 benchmarks/run.py
python3 benchmarks/score.py
```

`run.py` calls the Anthropic API once per (case, condition), saves each response under `results/`, and `score.py` produces the pass-rate table.

## Limitations

This is a small pilot, not a definitive study. To turn it into one:

- **N=5 is tiny.** Scale to N>=50 cases for any statistical signal. Bigger N reveals smaller deltas.
- **Two conditions, not four.** A clean experiment would also test (C) protocol-only, no threat and (D) silent retry, to isolate the threat from the protocol and from re-reading alone.
- **Single trial per case.** No variance estimation. Re-run each case N times to get confidence intervals.
- **One model.** Pilot uses Claude only. Test Codex, GPT-4, Gemini, etc. to see whether the framing generalizes.
- **Easy bugs.** Cases 1-5 are well-known footguns. To stress the framing, design bugs where the baseline (B) fails — that's the only difficulty range where blackmail framing has room to help.
