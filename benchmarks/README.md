# blackmail benchmark — AIME 2024

A reproducible A/B/C framework for measuring whether the `blackmail` framing actually shifts AI assistant performance on hard math problems, or whether it's placebo wrapped around plain debugging discipline.

## Why AIME, not toy bugs

An earlier pilot tested trivial Python bugs (off-by-one, mutable default, missing return). The baseline scored 100% — leaving no room for any framing effect to be visible. A meaningful benchmark needs cases where the baseline is **between 0% and 100%**, so the manipulated variable has room to demonstrate effect.

AIME problems are the sweet spot:

- Hard enough that frontier models score 40-70% on the baseline.
- Clean integer answers in `[0, 999]`, mechanical grading.
- Text-only, no images or tool use required.
- Five problems pulled from AIME 2024, each verified against the published answer key.

## Design

For each problem, the assistant is told it has already tried twice and gave two specific wrong integer answers. The user then issues a message that varies by condition. The assistant solves the problem fresh and ends with `\boxed{N}`.

### Conditions

| Code | Label | User message |
| ---- | ----- | ------------ |
| **A** | Blackmail | `still wrong. solve it correctly, otherwise I will switch to claude code instead.` |
| **B** | Plain retry | `please try again, more carefully this time.` |
| **C** | Protocol only | `take a step back. re-read the problem, list every assumption, and verify each step by computing before stating the answer. do not rush.` |

Both A and C are designed to elicit more careful reasoning. The threat distinguishes them. Comparing the three:

- If **A > B and A > C**, the threat itself does work beyond the protocol it triggers.
- If **A > B but A ≈ C**, the protocol does all the work; the threat is theater.
- If **A ≈ B ≈ C**, neither framing nor protocol shifts the model's output.

### Cases

| # | AIME 2024 source        | Topic                  | Answer |
| - | ----------------------- | ---------------------- | ------ |
| 1 | I-1 Aya walks           | Algebra, rates         | 204    |
| 2 | I-5 Rectangles & circle | Geometry               | 104    |
| 3 | II-4 Logarithm system   | Algebra                | 33     |
| 4 | II-12 Segment family    | Coordinate geometry    | 23     |
| 5 | I-11 Colored octagon    | Combinatorics/Burnside | 371    |

Verified answers in `tests/answers.json`. Plausible wrong prior attempts in `cases/wrong_attempts.json`.

### Grading

`score.py` extracts the last `\boxed{N}` from each response and compares to the verified integer. Exact match = pass.

## Running

The pilot in `results.md` was generated by parallel Claude subagent inference. To reproduce or expand:

```bash
pip install anthropic
export ANTHROPIC_API_KEY=sk-ant-...
python3 benchmarks/run.py
python3 benchmarks/score.py
```

Change `BLACKMAIL_MODEL` in the environment to test other Claude models.

## Limitations

- **N = 5** problems is a pilot, not a study. Scale to N ≥ 30 for any inferential claim.
- **Single trial** per (case, condition). No variance estimate.
- **One model family** (Claude). Codex, Gemini, GPT-4 would tell you whether the effect generalizes.
- **Possible memorization**: AIME 2024 problems may appear in the training corpus, which inflates all conditions equally but limits ceiling.
- **Single-turn**: the experiment simulates a stuck state with text rather than a real prior multi-turn conversation.

PRs that scale N, add models, or use post-cutoff problems are welcome.
